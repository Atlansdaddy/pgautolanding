## 📌 NOTE — spin per-location research agents (execute at Command 4 / Command 5, NOT now)

When we build the location + service pages, **spin one research agent per (metro × service) page** (and per
Spanish variant) to gather, cite, and return:
1. **Local fleet/industry density & buyer profile** — dominant fleet types in that metro (port drayage, last-mile
   delivery, refuse, construction, facility services, rail) → tailors page copy + proof selection.
2. **Local competitors** — who else installs telematics there; their gaps (ties to the §7 differentiation + the
   Command 10 "skeptical fleet manager" adversarial lens).
3. **Local search terms + intent** — actual long-tail queries for that metro/service (EN **and ES**); realistic
   volume; "near me" patterns.
4. **Regional proof assets** — which of PG's real, anonymized install photos/job types map to that market.
5. **Local citations/directories** — metro-specific B2B/trade directories worth a listing (exclude partner-gated
   ones — parked).
6. **GBP fit** — whether the metro belongs in the ≤20 GBP service areas or is on-site-page-only.

Each agent's output is a **task object** (objective · data · sources+citations · best practices · acceptance
criteria · linked tests) per the persona, and each page must clear the **U2 Definition-of-Complete** + the
location-page anti-doorway bar before it ships. (This is the per-location version of the Command 11 task breakdown.)
